# Contributing to Demo GitHub MCP

Thank you for your interest in contributing to our project! This guide will help you get started with contributing to our documentation.

## Documentation Style Guide

When contributing to documentation, please follow these guidelines:

### Writing Style

- **Be Clear and Concise**: Use simple, straightforward language
- **Use Active Voice**: Write "Click the button" instead of "The button should be clicked"
- **Be Consistent**: Follow the existing documentation structure and terminology
- **Include Examples**: Provide practical code examples where applicable
- **Keep it Updated**: Ensure documentation reflects the current state of the project

### Formatting Standards

- Use Markdown for all documentation files
- Include a table of contents for longer documents
- Use code blocks with appropriate syntax highlighting
- Add descriptive alt text for images
- Keep line length reasonable (80-100 characters when possible)

## How to Submit Documentation PRs

1. **Fork the Repository**: Create your own fork of the project
2. **Create a Branch**: Make a new branch for your documentation changes
   ```bash
   git checkout -b docs/your-feature-name
   ```
3. **Make Your Changes**: Edit or create documentation files
4. **Test Your Changes**: Preview your documentation locally
5. **Commit Your Work**: Write clear, descriptive commit messages
   ```bash
   git commit -m "docs: add API authentication guide"
   ```
6. **Push to Your Fork**: Push your branch to your GitHub fork
7. **Open a Pull Request**: Submit a PR with a clear description of your changes

### Pull Request Checklist

- [ ] Documentation is clear and easy to understand
- [ ] All code examples have been tested
- [ ] Spelling and grammar have been checked
- [ ] Links are working and point to the correct resources
- [ ] Changes follow the existing documentation structure

## Where to Ask Questions

If you have questions or need help:

- **GitHub Issues**: Open an issue with the `question` label
- **GitHub Discussions**: Join our community discussions (if enabled)
- **Pull Request Comments**: Ask questions directly in your PR

## Code of Conduct

Please note that this project follows our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## Recognition

All contributors will be recognized in our project documentation. Thank you for helping make our documentation better!
